Intégrations

IBM z/OS Intégration dans la marketplace Datadog

Vous pouvez désormais surveiller les mainframes IBM z/OS avec z/IRIS by Mainstorconcept, qui est disponible sur le Datadog Marketplace. L’offre de Mainstorconcept fournit des informations sur les mainframes IBM z/OS directement dans Datadog. Grâce à cette intégration, vous pouvez analyser les performances des instances DB2 exécutées sur z/OS via JDBC, surveiller l’utilisation du processeur dans le mainframe et obtenir des informations sur les batchs, l’activité des utilisateurs, la latence de z/OS Connect, etc.

Pour tester cette intégration gratuitement pendant 14 jours, visitez la vignette z/IRIS de Mainstorconcept sur le Datadog Marketplace, qui se trouve dans l’application Datadog sous « Intégrations » > « Marketplace ».

Pour plus de détails, vous pouvez consulter notre documentation, ainsi que le billet de blog Mainstorconcept z/IRIS de Datadog.

RUM

Surveiller les applications mobiles hybrides dans RUM

Real User Monitoring prend désormais en charge la surveillance des applications mobiles hybrides.

Cette nouvelle version permet à vos équipes d’ingénierie mobile de :

  • Suivre le parcours complet de l’utilisateur à travers les composants Web et natifs des applications mobiles.
  • Déterminer la cause première de la latence des pages Web ou des composants natifs dans les applications mobiles.
  • Aider les utilisateurs ayant des problèmes à charger des pages Web sur leurs appareils mobiles.

Pour commencer, mettez à niveau vers la dernière version des SDK : (iOS v1.10.0+, Android v1.12.0+, navigateur v4.7+) et ajoutez un extrait d’initialisation d’une ligne dans le SDK mobile. En savoir plus sur la fonctionnalité dans notre article de blog.

Extrayez événements et des données RUM agrégées à l’aide des API de recherche et d’analyse RUM

Vous pouvez désormais utiliser les API de recherche et d’analyse pour accéder aux événements RUM ainsi qu’aux données RUM agrégées. De plus, les utilisateurs peuvent facilement créer des appels d’API en transformant une requête intégrée dans l’explorateur RUM en une requête d’API en un clic avec le générateur de commandes cURL.

Pour générer automatiquement une requête API :

  • Accédez à RUM Explorer en accédant à UX Monitoring > Sessions > Replays
  • Créez la requête que vous souhaitez exécuter automatiquement
  • Cliquez sur le menu Exporter et sélectionnez Copier en tant que cURL.
  • Collez la commande cURL dans le terminal de ligne de commande de votre système d’exploitation pour y effectuer l’appel API ou copiez la requête API de la commande cURL dans un script ou un fichier de code.
  • Ajoutez votre clé API et votre clé d’application que les administrateurs peuvent trouver dans les Paramètres de l’organisation.
  • Effectuez l’appel d’API et analysez la réponse

Quels sont les cas d’utilisation typiques des API RUM ? Les API déverrouillent des cas d’utilisation, notamment :

  • Générez automatiquement des rapports à afficher dans vos tableaux de bord BI, par exemple :
    • Top liste des pages les plus visitées sur vos applications web
    • Nombre total de sessions utilisateur par zone géographique pour vos applications mobiles
    • Rapport sous forme de tableau du temps nécessaire à vos services d’API tiers pour répondre et analyser la violation de disponibilité.
  • Calculez automatiquement les KPI dans votre plate-forme de données, comme l’impact des incidents sur les utilisateurs en fonction de
    • La durée de l’incident extraite de votre outil de gestion des incidents
    • Le nombre d’utilisateurs sur le site Web extrait de RUM

En savoir plus sur nos API dans notre documentation et sur la façon de créer des requêtes à partir des données RUM dans l’explorateur dans notre documentation.

Synthetics

Tests de bout en bout avec CircleCI et Datadog Synthetics

Avec le nouvel orbe Synthetic CircleCI, vous pouvez déclencher vos tests synthétiques Datadog directement dans vos pipelines d’intégration et de livraison continues CircleCI, ce qui vous permet de bloquer le déploiement en cas d’échec des tests synthétiques.

Vous pouvez désormais vous assurer que les principaux flux de travail et systèmes de votre application ne sont pas affectés par les modifications de code tout au long du cycle de vie du développement. Avec le CircleCI Orb de Datadog Synthetics, vous pouvez :

  • Commencez avec les tests synthétiques sans avoir besoin de scripts d’intégration personnalisés, et ainsi :
  • Réduire le temps de création du pipeline CI/CD
  • Éliminez la prolifération des suites de tests et les silos de connaissances en utilisant les mêmes tests dans vos environnements de production et de CI

Commencez dès aujourd’hui avec Synthetics CircleCI orb dans vos pipelines CircleCI : https://docs.datadoghq.com/synthetics /cicd_integrations/circleci_orb/

Le Synthetics CircleCI Orb prend en charge notre Tunnel de test des synthétiques, vous permettant d’ouvrir une connexion sécurisée à l’infrastructure Datadog et de tester vos environnements locaux, le tout avec une seule ligne de code. Pour en savoir plus, consultez notre article de blog dédié : [https://www.datadoghq.com/blog/internal-application-testing-with-datadog/](https://www.datadoghq.com/blog/internal-application- test-avec-datadog/)

Faites évoluer votre installation avec nos emplacements privés synthétiques : https://docs.datadoghq.com/synthetics/private_locations

Consultez notre documentation et nos articles de blog récents pour en savoir plus sur Datadog Serverless Monitoring.

Contrôlez votre volume d’ingestion de traces avec la page de configuration Ingestion Control

La nouvelle page de contrôle de l’ingestion vous permet désormais d’obtenir un aperçu rapide de votre volume d’ingestion de span par rapport à votre allocation mensuelle d’ingestion.

Repérez les services responsables de la majeure partie de votre volume d’ingestion en un coup d’œil et explorez la configuration de n’importe quel service pour obtenir une visibilité sur la configuration de vos applications :

  • Comprenez quels mécanismes d’ingestion des bibliothèques d’agents et de traçage représentent la plus grande part du volume de portée grâce à la nouvelle dimension de raison d’ingestion.

  • Configurez les taux d’échantillonnage de vos services pour conserver ou supprimer un pourcentage de vos traces à partir du service

Essayez la nouvelle page Trace Ingestion Control dès aujourd’hui et consultez la documentation pour plus de détails.

Log Management

Annulez les réhydratations en cours

Log Rehydration analyse vos archives externes, telles qu’Amazon S3, Azure Storage et Google Cloud Storage, à la recherche de journaux historiques que vous souhaitez réintégrer dans Datadog à des fins de recherche et d’analyse.

Auparavant, vous deviez contacter le support Datadog pour annuler une réhydratation en cours en raison de la sélection d’une plage horaire incorrecte ou de fautes de frappe dans une requête de réhydratation. Désormais, vous pouvez annuler les réhydratations en cours directement sur la page Réhydrater à partir des archives.

Consultez notre documentation pour en savoir plus sur la réhydratation des journaux et [commencez à l’utiliser dès aujourd’hui]. (https://app.datadoghq.com/logs/pipelines/historical-views).

Métriques d’utilisation estimées pour les pipelines de log

Les métriques d’utilisation estimées pour Log Pipelines sont désormais disponibles. Analysez la quantité de données circulant dans vos pipelines, lesquelles sont porteuses de charge et recherchez le trafic non analysé pour améliorer le traitement des log. Obtenez une image complète de l’utilisation de vos pipelines de log, des volumes d’ingestion et de l’échantillonnage d’index à l’aide du nouveau tableau de bord d’utilisation. Consultez notre documentation pour en savoir plus sur ces fonctionnalités.

Attribuez facilement des utilisateurs aux bons groupes d’autorisations grâce aux rôles personnalisés

Les rôles personnalisés vous permettent de définir des groupes d’autorisations pour vos utilisateurs, ce qui vous permet de les gérer et de les modifier facilement. Utilisez des rôles personnalisés pour spécifier les droits de modification, limiter l’accès à certains ensembles de log, et bien plus encore.

Notre dernière version améliore l’interface des rôles, ce qui facilite l’affichage et la navigation dans la liste des autorisations disponibles. Nous avons également facilité la création de rôles à l’aide du clonage de rôle et de modèles de rôle prêts à l’emploi. Enfin, nous avons renforcé le contrôle en divisant l’autorisation “Accès privilégié” ou “administrateur” en un [ensemble d’autorisations] plus précis(https://docs.datadoghq.com/account_management/rbac/permissions/).

Lisez notre documentation pour en savoir plus et commencer à utiliser les rôles client dès aujourd’hui !

Monitors

Les notifications par e-mail des moniteurs font peau neuve

Les moniteurs garantissent que vos équipes sont alertées en cas de problèmes sur vos systèmes. Il est essentiel de pouvoir comprendre rapidement le problème et les prochaines étapes de dépannage pour garantir que vos équipes maintiennent un MTTR bas. Les notifications par e-mail de Monitor ont été repensées pour vous permettre de comprendre ce qui ne va pas en quelques secondes.

Vous pouvez en savoir plus sur les notifications de surveillance dans la documentation.

Recevez des notifications push mobiles pour les incidents

Lorsqu’un incident se produit, vous devez être en mesure de rester au courant, où que vous soyez. Pour vous tenir informé des incidents lors de vos déplacements, l’application mobile Datadog peut vous informer des nouveaux incidents et vous tenir au courant jusqu’à leur résolution.

Avec les notifications push dans l’application mobile Datadog, vous pouvez :

  • Être automatiquement alerté des incidents où vous êtes le commandant de l’incident.
  • Suivre les incidents de la création à la résolution, en restant informé à mesure que l’incident change.
  • Rejoindre le chat pour un incident, directement à partir de la notification d’incident sur votre écran de verrouillage.

Téléchargez l’application mobile Datadog (disponible sur iOS et Android) pour commencer. Pour en savoir plus sur les règles de notification des incidents, consultez notre documentation.
